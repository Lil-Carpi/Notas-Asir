abrir el dc se ve 172.0.0.1 para resolver dns

Para que los clientes usen el dns del server:
- cambiamos el dns del cliente a la ip fija del server
- se prueba el dns con dnslookup



2 tipos de usuarios
- Usuario local
- Usuario global
	El que se crea en el AD

Diferencias entre amas: el global se abre en todas las máquinas del dominio

Directiva de grupo (gpo) en el server no se puede entrar a los usuarios del dominio

Se han de incorporar los clientes a dominio desde cliente
- se cambia el hostname
- Se une a dominio cambiando el WORKGROUP al dominio del addc.
- Se pedira el usuario y contraseña del admin
- Ahora se puede loguear con el usuario global
