## 1.1 Ficheros y directorios
### 1.1.1 Nomenclatura

En los sistemas MS-DOS, no se distinguen entre mayúsculas y minúsculas. `Documento.txt` y `documento.txt` son exactamente el mismo fichero. ¿No se permiten caracteres como `\ / : * ? " < > |`.

En los sistemas Unix, **Sí** se distinguen entre mayúsculas y minúsculas. `Documento.txt`, `documento.txt` y `DOCUMENTO.TXT` son tres ficheros totalmente distintos. El unico caracter prohibido es la `/` (reservada para directorios) y el carácter nulo.

### 1.1.2 Tipos de ficheros
Los sistemas unix se basan fuertemente en las **extensiones** (`.exe`, `.doc`, `.txt`) para saber qué programa abre cada archivo.

En los sistemas Unix, todo es un archivo. Las extensiones son opcionales y sirven más para que el usuario pueda interpretar que es el fichero con el que se está tratando. Linux utiliza los **Metadatos** (el comando `file` te dice qué es un archivo realmente) y divide los archivos en: ordinarios, directorios, enlaces y archivos de dispositivo.

### 1.1.3 Rutas
La ruta absoluta es aquella que indica la posicion exacta desde la raiz del sistema.
En los sistemas MS-DOS, se empieza con la letra de la unidad (`C:\Users\Usuario\Documents`).
En los sistemas Unix, se empieza desde la raíz `/` (`/home/usuario/documentos`).

Las rutas relativas son aquellas que indican la posición partiendo desde donde estás metido ahora mismo.
Un punto (`.`) significa "directorio actual".
Dos puntos (`..`) significa "subir al directorio padre".

## 1.2 Estructura de directorios
### 1.2.1 Estructura de directorios de Windows.
Windows utiliza la estructura basada en **unidades de disco** (C:, D:, etc.). Las carpetas principales son:
`C:\Windows`: Archivos del propio sistema operativo
`C:\Program Files` (o Archivos del programa): Donde se instalan los softwares.
`C:\Users`: Carpetas personales de los usuarios.

### 1.2.2 Estructura de directorios de Linux
Linux utiliza un arbol unico que cuelga de la raiz `/`. No hay "unidades C o D"; los discos se "montan" dentro de este árbol.

`/bin` y `/sbin`: Comandos y binarios esenciales del sistema.
`/home`: Carpetas personales de los usuarios (equivalente a `C:\Users`).
`/root`: Carpeta personal de root (administrador/superusuario).
`/etc`: Archivos de configuracion del sistema.
`/var`: Archivos variables (logs, bases de datos, ficheros web, etc.).

## 1.4 Permisos
#### 1.4.1 Permisos en Windows
Los sistemas Windows utilizan **ACLs (Listas de Control de Acceso)**. Son muy granulares. Puedes definir de forma específica qué usuario o grupo tiene los permisos de *Lectura, Escritura, Ejecución, Modificación* o *Control Total* haciendo `clic derecho -> Propiedades -> Seguridad.`

#### 1.4.2 Permisos en Linux
Se dividen en tres grupos de acción (read, write. execute) para los tres tipos de usuarios: **Propietario** (u, valor octal `4`), **Grupo** (g, valor octal  `2`) y **Otros** (o, valor octal `1`).

Se representan de forma simbólica (ej: `rwxr-xr--`) o numérico (ej: `755`). 
Se basan en Octetos, del `0` (nada), hasta el 7 (todos los permisos) Los permisos de valor 
numérico se calculan sumando la cantidad de permisos que se le dan a los tres tipos de usuario. 

Comandos:
`chmod`: Cambia los permisos (ej: `chmod 755 archivo`).
`chown`: Cambia el propietario del archivo (ej: `chown usuario:grupo fichero`)
	Para `chown`, podemos indicar tambien el grupo propietario del fichero. Se hace separando el usuario con `:`. Ej: `chown :grupo fichero`

##### 1.4.2.1 Permisos especiales en Linux
Como he explicado antes, normalmente conocemos los permisos estandar: Lectura (`r`), Escritura (`w`) y Ejecución (`x`), pero en Linux existen **3 bits especiales** que se colocan justo delante de los permisos normales cuando usamos la notacion numerica, o modificamos la `x` en la notacion simbolica.

Estos bits son: **SUID**, **SGID** y el **Sticky bit** (`t`).
###### 1.4.2.1.1 SUID (Set User ID)
Valor numerico: `4`
Cuando un archivo ejecutable tiene este bit activo, cualquier usuario que lo ejecute **no lo ejecutara con sus propios permisos, sino con los permisos del propietario del archivo** (que suele ser `root`).
Un ejemplo de esto aplicado es el comando `/usr/bin/passwd` (`passwd` de toda la vida, para cambiar la contraseña). Para cambiar la contraseña, el sistema tiene que escribir en el archvo `/etc/shadow`, pero este archivo solo lo puede modificar `root`. Gracias al SUID, cuando tú ejecutas `passwd`, el proceso corre temporalmente con privilegios de superusuario para poder guardar tu contraseña, y luego se cierra.
Esto se ve cuando se modifica la `x` del propietario. Se representa con una  `s` minuscula (o `S` mayuscula si el archivo original no tenía permisos de ejecucion).
	- `rwsr-xr-x` (Tiene SUID y ejecucion)

##  1.5 Enlaces
En Windows, el enlace mas comun es el **Acceso directo** (`.lnk`), que es simplemente un archivo que apunta a otro- También existen los *Symlinks* por línea de comandos (`mklink`), pero se usan menos a nivel de usuario.

En Linux existen dos tipos fundamentales creados con el comando `ln`:
**Enlaces blandos o simbólicos (*Symbolic Links - `ln -s`*)**: Son como los accesos directos. Si borras el original, el enlace se rompe ("queda huérfano").
**Enlaces duros (*Hard Links - `ln`*)**: Apuntan directamente al mismo sector del disco (mismo *inodo*). Si borras el original, el archivo sigue existiendo mientras quede al menos un enlace duro.

## 1.6 Búsqueda de información
### 1.6.1 Windows
De manera grafica: El explorador de archivos tiene una barra de búsqueda que indexa el contenido.
Linea de comandos (cmd): Comando `findstr` para buscar texto dentro de los archivos.

### 1.6.2 Linux
`find`: Busca **archivos** por nombre, tamaño, fecha, etc. (ej: `find /home -name "*.txt"`).
`grep`: Busca **texto dentro** de los archivos (ej: `grep "error" sistema.log`).

## 1.7 Identificación del programa instalado
### 1.7.1 Windows
**Gráfico**: Configuración -> Aplicaciones -> Aplicaciones instaladas, o Panel de Control -> Programas y características.
PowerShell: `Get-WmiObject -Class Win32_Product` O usando tools como `winget list`.

### 1.7.2 Linux
Depende de la familia de la distro:
**Debian / Ubuntu / Mint (paquetes .deb)**: `dpkg -l` o `apt list --installed`.
**Red Hat / Fedora / CentOS (paquetes .rpm)** `rpm -qa` o `dnf list installed`.
