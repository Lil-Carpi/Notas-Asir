## 2.1 Distribuciones de disco duro
#### 2.1.1 Particiones de disco duro
Una particion es una division logica dentro de un disco fisico. El sistema operativo trata cada particion como si fuera un disco independiente.

### 2.1.2 Nomenclatura de los discos duros
La forma de nombrar los discos fisicos ys sus particiones son diferentes en cada sistema operativo:
En Windows, no se expone el nombre del disco fisico al usuario; se les asignan las **letras de unidad (`C:`, `D:`, `E:`)**.

En linux, todo es un archivo en `/dev`.
	Discos HDD: `hda` (primer disco), `hdb` (segundo disco).
	Discos SATA/SST/USB: `sda` (primer disco), `sdb` (segundo disco).
	Discos NVMe (M.2): `nvme0n1` (primer disco), `nvme1n1` (segundo disco).
	**Particiones**: Se les añade un numero al final. La primera partición del primer disco SATA es `sda1`; la segunda es `sda2`.

### 2.1.3 Ventajas de usos de particiones.
**Soporte para diversos sistemas operativos**: Permite el *Dual Boot* (instalar dos sistemas operativos en varias particiones).

**Uso de diversos sistemas de ficheros**: Cada particion puede tener un formato distinto (ej: una en NTFS y otra en ext4).

**Separacion del OS y datos**: Si Windows se rompe, reinstalas el sistema en `C:\` sin tocar tus 
datos personales que "deberian" estar a salvo en `D:\`. (En Linux es comun separar `/` de `/home`)

**Protecion ante errores**: Si se corrompe el sistema de archivos de una particion, las demas suelen quedar intactas.

**Seguridad**: Puedes aplicar restricciones estrictas (como "solo lectura") a una particion especifica.

**Copias de seguridad**: Es mas facil clonar o hacer backup de una particion pequña y critica que de un disco entero de 2 TB.

### 2.1.4 Tablas de particiones
Define como se guarda la informacion para las particiones en el disco. Hay dos estandares:
**MBR (Master Boot Record)**: Estandar antiguo.
	Se permiten maximo 4 particiones primarias (o 3 primarias y 1 extendida que dentro contiene particiones lógicas).
	Tamaño maximo del disco: 2 TB.
**GPT (GUID Partition Table)**: El estandar moderno (requiere BIOS UEFI).
	Practicamente sin limite de particiones (hasta 128 en Windows).
	Soporta discos de hasta 9.4 Zettabytes.
	Guarda copias de seguridad en la tabla al final del disco por seguridad.

## 2.2 Sistemas de ficheros
El sistema de archivos es lo que utiliza un sistema operativo para organizar, guardar y encontrar los archivos de dentro de una particion.

### 2.2.1 Topologias de sistemas de ficheros
Los modernos utilizan **Journaling (Bitácora)**. Es un mecanismo que registra las operaciones antes de escribirlas. Si hay un apagon, el sistema lee el *journal* y evita que el disco se corrompa.

### 2.2.2 Sistemas de ficheros comunes
**Windows**
	**NTFS**: El estandar actual. Soporta compresion, cifrado (BitLocker), permisos avanzados y archivos gigantes.
	**FAT32**: Antiguo pero aun se usa. No soporta archivos de mas de 4 GB. Ideal para USBs viejos.
	**exFAT**: Sustituto de FAT32 para pendrives y tarjetas SD. Quita el limite de los 4GB y es compatible con Windows, Mac y Linux.
**Linux**
	**ext4**: El estandar por defecto de casi todo Linux. Fiable, rapido y con *Journaling*.
	**Brtfs / XFS**: Sistemas avanzados para servidores o almacenamiento masivo con funciones de instantaneas (snapshots). Las distribuciones *Enterprise* suelen llevar este tipo de sistema de ficheros, como por ejemplo **Red Hat Enterprise Linux**.

## 2.3 Manipulacion de particiones
### 2.3.1 Por medio de entorno grafico
**Windows**: Comando `diskpart`. Es un entorno interactivo potente para gestionar los discos. Va por selecciones.
**Linux**: `fdisk` / `gdisk`: Para crear y borrar particiones MBR o GPT.
	`mkfs.ext4 /dev/sda1` o `mkfs.vfat /dev/sda2`: Para formatear (crear el sistema de archivos).

## 2.4 Montaje y desmontaje de particiones
Para que un sistema operativo pueda usar una particion, debe estar montada (conectada al sistema).
